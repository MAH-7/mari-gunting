## üìã Migration Pattern (Copy-Paste Template)

### Step 1: Update State Variables

**BEFORE:**
```typescript
const [imageUri, setImageUri] = useState<string | null>(null);
const [imageUrl, setImageUrl] = useState<string | null>(null);
const [uploading, setUploading] = useState(false);
```

**AFTER:**
```typescript
const [imageUri, setImageUri] = useState<string | null>(null);
const [isLoadedFromProgress, setIsLoadedFromProgress] = useState(false);
const [uploading, setUploading] = useState(false);
```

**Changes:**
- ‚ùå Remove `imageUrl` state (uploaded URLs)
- ‚úÖ Add `isLoadedFromProgress` flag

---

### Step 2: Update `loadProgress` Function

**BEFORE:**
```typescript
const loadProgress = async () => {
  const progress = await service.getProgress();
  if (progress.data) {
    setImageUrl(progress.data.imageUrl);
  }
};
```

**AFTER:**
```typescript
const loadProgress = async () => {
  const progress = await service.getProgress();
  if (progress.data) {
    // Load URLs as display URIs
    setImageUri(progress.data.imageUrl);
    setIsLoadedFromProgress(true);
    console.log('‚úÖ Progress loaded - images already uploaded');
  }
};
```

---

### Step 3: Update `pickImage` / `takePhoto` Functions

**BEFORE:**
```typescript
const pickImage = async () => {
  const result = await ImagePicker.launchImageLibraryAsync({...});
  
  if (!result.canceled) {
    const uri = result.assets[0].uri;
    
    // Upload immediately ‚ùå
    setUploading(true);
    const url = await uploadOnboardingImage(uri, ...);
    setImageUri(uri);
    setImageUrl(url);
    setUploading(false);
  }
};
```

**AFTER:**
```typescript
const pickImage = async () => {
  const result = await ImagePicker.launchImageLibraryAsync({...});
  
  if (!result.canceled) {
    const uri = result.assets[0].uri;
    
    // Store locally only ‚úÖ
    setImageUri(uri);
    console.log('‚úÖ Image stored locally - will upload on submit');
  }
};
```

**Key Changes:**
- ‚ùå Remove upload logic
- ‚ùå Remove `setUploading(true/false)`
- ‚úÖ Just set local URI

### Step 4: Add Batch Upload Function

```typescript
const uploadAllImages = async (): Promise<{
  imageUrl: string;
  // Add other image URLs here
} | null> => {
  try {
    console.log('üì§ Starting batch upload...');
    setUploading(true);

    // Upload images in parallel
    const uploadTasks = [
      uploadOnboardingImage(imageUri!, 'bucket-name', user?.id || 'temp', `image_${Date.now()}.jpg`),
      // Add more upload tasks
    ];

    const [imageUrl, /* other URLs */] = await Promise.all(uploadTasks);

    if (!imageUrl) {
      console.error('‚ùå Upload failed');
      return null;
    }

    console.log('‚úÖ All images uploaded successfully!');
    return { imageUrl, /* other URLs */ };
  } catch (error) {
    console.error('‚ùå Batch upload error:', error);
    return null;
  } finally {
    setUploading(false);
  }
};
```

### Step 5: Update `handleSubmit` / `handleContinue`

**BEFORE:**
```typescript
const handleContinue = async () => {
  if (!validateForm()) return;

  setLoading(true);
  
  const data = {
    imageUrl,  // Already uploaded
  };

  await service.saveProgress(data);
  router.push('/next-step');
  setLoading(false);
};
```

**AFTER:**
```typescript
const handleContinue = async () => {
  if (!validateForm()) return;

  setLoading(true);

  // If loaded from progress, skip upload
  if (isLoadedFromProgress) {
    console.log('‚úÖ Using previously uploaded data');
    router.push('/next-step');
    return;
  }

  // Upload all images
  console.log('üöÄ Uploading all images...');
  const uploadedUrls = await uploadAllImages();

  if (!uploadedUrls) {
    Alert.alert('Upload Failed', 'Please check your connection and try again.');
    return;
  }

  // Save with uploaded URLs
  const data = {
    ...uploadedUrls,
  };

  await service.saveProgress(data);
  console.log('üéâ Progress saved successfully!');
  
  router.push('/next-step');
  setLoading(false);
};
```

## üìä Performance Comparison

### Before (Immediate Upload):
- User selects 5 photos ‚Üí 5 separate uploads ‚Üí 5-10 seconds total
- User changes 2 photos ‚Üí 2 more uploads ‚Üí 10-15 seconds total
- **Total: 7 uploads, 15+ seconds**

### After (Staged Upload):
- User selects 5 photos ‚Üí Instant (no upload)
- User changes 2 photos ‚Üí Instant (no upload)
- User clicks Continue ‚Üí 5 parallel uploads ‚Üí 3-5 seconds
- **Total: 5 uploads, 5 seconds**

**Savings:**
- ‚ö° **66% faster** (5s vs 15s)
- üíæ **28% less storage** (no orphaned files)
- üì° **Better UX** (instant feedback)

## üî• Quick Copy-Paste Checklist

When refactoring a screen:

1. ‚úÖ Remove `*Url` state variables
2. ‚úÖ Add `isLoadedFromProgress` state
3. ‚úÖ Update `loadProgress()` to set URIs from saved URLs
4. ‚úÖ Remove upload logic from `pickImage()` / `takePhoto()`
5. ‚úÖ Create `uploadAll*()` batch upload function
6. ‚úÖ Update `handleSubmit()` to call batch upload
7. ‚úÖ Update validation to check local URIs, not URLs
8. ‚úÖ Add info banner (optional)
9. ‚úÖ Update any display logic from `*Url` to `*Uri`
10. ‚úÖ Test thoroughly!
